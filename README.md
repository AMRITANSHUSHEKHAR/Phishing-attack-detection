# Phishing-attack-detection
1.1	 Problem statement
Recent studies have shown that phishers are using phishing kits to deploy phishing attacks faster, easier and more massive. Detecting phishing kits in deployed websites might help to detect phishing campaigns earlier. This project addresses the challenges of Machine learning based system for phishing attack detection, this project aims to develop a solution that not only enhances the accuracy of model.
1.2	 Introduction 
Phishing attacks have become anxiety for the cyber world. It causes enormous problems for privacy and financial issues of internet users. Scammers, namely fishers, create false websites to feel and look like a genuine to deceive the people. They send emails to steal the identity of legitimate users. As technology advances, so do the tactics employed by malicious actors seeking to exploit vulnerabilities in human behaviour and online systems. Traditional methods of phishing detection, such as rule-based systems and signature-based approaches, often struggle to keep pace with the rapidly changing nature of phishing attacks. In response to this challenge, machine learning (ML) has emerged as a powerful tool for enhancing the detection capabilities of anti-phishing systems.

This report explores the application of machine learning techniques to the realm of phishing attack detection. We aim to develop a system that can adapt to the dynamic nature of phishing attacks and provide a robust defence against increasingly threats.

1.3	 Objectives
•	Data Acquisition: Retrieve and load datasets (train and test sets) to analyse different legitimate and phishing URLs. Understand the structure and characteristics of the data.
•	Data Pre-processing: Clean and pre-process the dataset containing datasets. Applying data balancing techniques.
•	Exploratory Data Analysis (EDA): Explore the distribution of dataset in the training set. Visualize the key features and patterns within the data.
•	Model Training: Implement different models like Decision Tree, Random Forest, XGBoost and SVM to the dataset. Train the model on the pre-processed training dataset.
•	Model Evaluation: Assess the model's performance using cross-validation on the test set. Measure accuracy to gauge the effectiveness of sentiment prediction.
•	Model Persistence: Save the trained model for future use. Establish a mechanism for loading the saved model for detection of phishing attack.
